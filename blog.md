## Architecture Update — 2026-04-25

The default programme has been revised from the v2/v3 micro-pass model to a v4 rewrite-wave model. The old one-clause, one-dimension loop remains available as surgical mode for high-risk clauses, but normal policy improvement now starts with a compact packet for a coherent topic cluster.

The new role split is architect, planner, author, editor, and verifier. Persistent maps have been added under `maps/` so future runs can select and verify policy regions without rediscovering the whole repository each time. The outer loop now uses a three-tier planning structure: `policy-architecture-strategy.md`, `structural-backlog.md`, and `rewrite-wave-queue.md`. Full graph-based structural consolidation remains available when more standards are added.
